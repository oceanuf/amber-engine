## 🎯 核心宗旨

**真金不怕火炼，实战是检验真理的唯一标准**

演武场是琥珀引擎的"实战检验场"，通过真实市场环境验证算法有效性，实现：
1. **算法进化**: 动态权重调整，让表现优秀的算法获得更大话语权
2. **标的筛选**: 公平竞技，优胜劣汰，构建最优投资组合
3. **系统自愈**: 通过实战反馈优化上层感知层（哨兵）和搜索层（探针）

## 🏗️ 系统架构

### 三层协同架构

| 层级 | 模块 | 职能 | 输出物 |
|------|------|------|--------|
| **感知层** | 哨兵 (Sentry) | 市场监控，异常检测 | 候选标的池 |
| **搜索层** | 探针 (Probe) | 深度挖掘，信息验证 | 推荐标的列表 |
| **评委层** | G1-G11算法 | 多维度独立评分 | 初始评分矩阵 |
| **决策层** | 演武场引擎 | 综合评分，权重计算 | 入选/淘汰决策 |
| **执行层** | 虚拟基金引擎 | 实际交易执行 | 持仓/交易记录 |
| **反馈层** | 数据桥接器 | 绩效分析，权重调整 | 算法权重更新 |

### 数据流向
```
哨兵 → 探针 → G1-G11评分 → 演武场决策 → 虚拟基金执行 → 绩效反馈 → 权重调整
```

## 📊 标的评分与入选规则

### 1. 初始评分计算

**公式**: 
```
S_initial = Σ (G_i_score * G_i_weight)
```
其中：
- `G_i_score`: 算法i对标的的评分 (0-100分)
- `G_i_weight`: 算法i的当前权重 (初始值: 1/11 ≈ 0.0909)

**评分标准**:
- 每个算法独立评分，基于各自逻辑（价值、动量、统计、宏观等）
- 评分范围: 0-100分，整数
- 评分依据: 当日市场数据（收盘后计算）

### 2. 入选阈值

| 队列 | 阈值条件 | 初始资金 | 最大数量 | 权重影响 |
|------|----------|----------|----------|----------|
| **演武场1队** | `S_initial ≥ 70` | 100,000元 | 30支 | ✅ 影响算法权重 |
| **演武场2队** | `60 ≤ S_initial < 70` | 50,000元 | 20支 | ❌ 不影响权重 |
| **观察池** | `S_initial < 60` | 0元 | 不限 | ❌ 不影响权重 |

### 3. 每日调仓流程

**时间**: 交易日收盘后 (15:00 GMT+8)
**步骤**:
1. **计算评分**: 对所有候选标的进行G1-G11算法评分
2. **排名筛选**: 按`S_initial`降序排列
3. **队列分配**:
   - 前30名且`S_initial ≥ 70`: 进入/保留在1队
   - 第31-50名且`60 ≤ S_initial < 70`: 进入/保留在2队
4. **资金分配**: 
   - 新入选标的获得相应初始资金
   - 淘汰标的平仓，资金回收
5. **交易执行**: 调用虚拟基金引擎执行买卖

## 🔄 算法权重动态调整机制

### 1. 权重计算原则

**核心思想**: "谁能帮我赚到钱，谁就有更大话语权"

**权重更新频率**: 每日收盘后，与调仓同步

### 2. 表现度量指标

**算法i的表现度量**:
```
performance_i(t) = (1 + avg_excess_return_i) ^ 30 - 1
```
其中：
- `avg_excess_return_i`: 算法i推荐标的的平均日超额收益（过去30天）
- **超额收益** = 标的收益率 - 沪深300(510300)同期收益率

### 3. 半衰期衰减

**指数加权移动平均 (EWMA)**:
```
weighted_performance_i(t) = α * performance_i(t) + (1-α) * weighted_performance_i(t-1)
```
- `α = 0.1` (对应半衰期约6.5天)
- 确保近期表现权重更高，防止"路径依赖"

### 4. 权重更新公式

**原始权重**:
```
raw_weight_i(t) = weighted_performance_i(t) / Σ weighted_performance_j(t)
```

**平滑处理** (防止剧烈波动):
```
G_i_weight(t) = 0.8 * G_i_weight(t-1) + 0.2 * raw_weight_i(t)
```

**归一化**:
```
G_i_weight(t) = G_i_weight(t) / Σ G_j_weight(t)
```

### 5. 权重边界保护

- **最低权重**: 0.01 (1%) - 防止完全失效的算法被彻底遗忘
- **最高权重**: 0.30 (30%) - 防止单一算法垄断决策
- **新算法保护期**: 新引入算法前30天权重固定为平均值

## 🏆 演武场管理规则

### 1. 队列管理

**演武场1队 (正式队)**:
- 最大持仓: 30支
- 初始资金: 每支100,000元
- 调仓频率: 每日
- 绩效影响: ✅ 影响算法权重

**演武场2队 (替补队)**:
- 最大持仓: 20支  
- 初始资金: 每支50,000元
- 调仓频率: 每周一
- 绩效影响: ❌ 不影响算法权重

### 2. 晋升/降级机制

**晋升条件** (2队 → 1队):
1. 标的`S_initial ≥ 70`
2. 1队有空缺位置
3. 在2队中连续3天排名前5

**降级条件** (1队 → 2队):
1. 标的`S_initial < 70`
2. 在1队中连续3天排名后5
3. 或触发强制止损

**淘汰条件** (2队 → 观察池):
1. 标的`S_initial < 60`
2. 在2队中连续5天排名后3

### 3. 资金管理

**总资金池**: 5,000,000元 (500万)
**分配比例**:
- 1队最大占用: 3,000,000元 (30支 × 10万)
- 2队最大占用: 1,000,000元 (20支 × 5万)
- 现金储备: 1,000,000元 (20%)

**动态调整**:
- 当1队有空缺时，优先使用现金储备
- 现金不足时，从2队抽调资金
- 总资金使用率超过80%时，暂停新标的人选

## ⚖️ 交易规则

### 1. 买入规则

**时机**: 入选当日收盘价
**价格**: 收盘价 + 滑点 (0.05%)
**仓位**: 固定金额 (1队: 10万, 2队: 5万)
**锁仓期**: 30天 (继承现有虚拟基金引擎规则)

### 2. 卖出规则

**止盈策略** (移动止盈):
1. **第一止盈点**: 收益率 ≥ +20% → 卖出50%仓位
2. **第二止盈点**: 收益率 ≥ +30% → 卖出剩余仓位
3. **回落止盈**: 从最高点回落 ≥ 5% → 卖出全部仓位

**止损策略**:
1. **单日止损**: 单日亏损 ≥ 10% → 强制平仓
2. **累计止损**: 累计亏损 ≥ 15% → 强制平仓
3. **时间止损**: 持有60天仍无盈利 → 平仓

**调仓卖出**:
1. 因评分不足被移出演武场
2. 被更优秀的标的替换

### 3. 交易成本

**磨损成本模型**:
- 佣金: 0.05% (双向)
- 印花税: 0.1% (卖出单边)
- 滑点: 0.05% (双向)
- **总交易成本**: 约0.25%每笔完整交易

**绩效计算**: 所有收益率均为扣除交易成本后的净收益率

## 🚨 风险控制规则

### 1. 全局熔断 (裁判长一票否决权)

**触发条件** (任一条件满足):
1. **市场恐慌**: 全市场跌停家数 > 1000
2. **指数暴跌**: 沪深300单日跌幅 > -5%
3. **流动性危机**: 两市成交额 < 3000亿
4. **政策风险**: 重大政策利空公告

**熔断动作**:
1. **暂停新开仓**: 所有新标的人选暂停
2. **持仓保护**: 现有持仓暂不执行止盈止损
3. **观察期**: 熔断持续至条件解除后2个交易日
4. **恢复评估**: 市场稳定后重新评估所有持仓

### 2. 算法末位淘汰

**淘汰条件**:
```
连续60个交易日，算法推荐的高分标的(评分≥80)平均超额收益 ≤ 0
```

**淘汰动作**:
1. **权重归零**: 算法权重强制设为0.01
2. **观察期**: 进入6个月观察期
3. **复活条件**: 在观察期内，算法在替补队的推荐标的实现正超额收益
4. **永久淘汰**: 观察期结束后仍无改善，从系统中移除

### 3. 集中度风险控制

**个股集中度**:
- 单只股票仓位 ≤ 总资产的20%
- 单只股票亏损 ≤ 总资产的5%

**行业集中度**:
- 单一行业配置 ≤ 总资产的30%
- 至少覆盖5个不同行业

**流动性风险**:
- 平均日成交额 ≥ 持仓市值的10%
- 小市值股票 (< 100亿) 仓位 ≤ 总资产的15%

## 📈 绩效评估体系

### 1. 基准对比

**主要基准**: 沪深300指数 (510300)
**对比指标**:
1. **超额收益 (Alpha)**: 组合收益 - 基准收益
2. **信息比率**: 超额收益 / 跟踪误差
3. **胜率**: 战胜基准的天数比例

### 2. 算法贡献度分析

**贡献度分解**:
```
组合超额收益 = Σ (算法i权重 × 算法i贡献的超额收益)
```

**算法i的贡献**:
1. **选股能力**: 推荐标的本身的超额收益
2. **权重影响**: 算法权重变化带来的贡献
3. **交互效应**: 多算法共振的协同效应

### 3. 风险调整后收益

**主要指标**:
1. **夏普比率**: (组合收益 - 无风险收益) / 波动率
2. **最大回撤**: 历史最大累计亏损
3. **卡尔玛比率**: 年化收益 / 最大回撤
4. **索提诺比率**: (组合收益 - 无风险收益) / 下行风险

### 2. 执行流程

**每日收盘后 (15:10 GMT+8)**:
1. **数据准备**: 获取当日收盘数据、算法评分
2. **权重计算**: 更新G1-G11算法权重
3. **综合评分**: 计算所有标的`S_initial`
4. **队列调整**: 确定晋升/降级/淘汰
5. **交易指令**: 生成买卖指令
6. **执行交易**: 调用虚拟基金引擎
7. **生成反馈**: 创建反馈文件
8. **报告生成**: 生成每日报告

### 3. 监控与告警

**监控指标**:
1. **执行成功率**: 每日调仓执行成功率 ≥ 95%
2. **数据新鲜度**: 数据延迟 ≤ 1小时
3. **权重稳定性**: 单日权重变化 ≤ 10%
4. **资金利用率**: 70% ≤ 资金使用率 ≤ 90%

**告警阈值**:
- 执行失败连续3次 → 紧急告警
- 算法权重剧烈波动 → 警告
- 资金使用率超过95% → 警告
- 全局熔断触发 → 紧急告警
