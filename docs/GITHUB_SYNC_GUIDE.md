# 🏛️ GitHub同步使用指南 - 基于法典V5.0

## 概述

本指南基于[2614-001号]档案馆单体仓库同步法典和琥珀引擎共同记忆核心记忆法典(V5.0)，为"GitHub同步"命令提供标准化操作流程。

## 核心原则

### 1. 认知对齐
每次同步前必须确认双方对以下内容认知一致：
- 环境配置状态
- 法典版本要求
- 架构合规状态
- 同步工具选择

### 2. 法典合规
所有同步操作必须符合：
- **[2614-001号]**: 同步法典
- **法典V5.0**: 共同记忆法典
- **永久禁令**: 三条硬性规定

### 3. 高效执行
通过标准化流程减少沟通成本，确保一次成功。

## 命令格式

### 基础命令
```
GitHub同步
```
**执行**: 标准日常同步，使用 `[SYNC]` 前缀

### 带描述的命令
```
GitHub同步 [描述内容]
```
**执行**: 使用 `[SYNC]: 描述内容` 作为提交信息

### 带前缀的命令
```
GitHub同步 [前缀]: [描述内容]
```
**执行**: 使用指定的前缀，如 `[ARCH]: 描述内容`

### 可用前缀
- `[ARCH]`: 架构变更
- `[VAULT]`: 金库内容更新
- `[DOC]`: 文档更新
- `[SEC]`: 安全相关
- `[DATA]`: 数据更新
- `[SYNC]`: 日常同步 (默认)

## 错误处理

### 常见问题及解决方案

#### 问题1: 环境配置缺失
**症状**: `~/.amber_env` 不存在或配置错误
**解决**: 
```
请确保 ~/.amber_env 文件存在并包含：
export GITHUB_TOKEN="your_token"
export GITHUB_REPO="username/repo"
```

#### 问题2: 权限不足
**症状**: GitHub API返回403错误
**解决**:
1. 检查令牌有效期
2. 确认令牌有仓库写入权限
3. 更新 `~/.amber_env` 中的令牌

#### 问题3: 冲突检测
**症状**: `git pull` 报告冲突
**解决**:
```
检测到冲突，执行冲突解决流程：
1. 拉取最新更改: git pull --rebase origin main
2. 解决冲突文件
3. 重新执行同步
```

#### 问题4: 推送保护阻止
**症状**: GitHub秘密扫描阻止推送
**解决**:
```
GitHub推送保护检测到敏感信息：
1. 检查并删除包含密钥的文件
2. 使用 git commit --amend 修改提交
3. 重新推送
```
