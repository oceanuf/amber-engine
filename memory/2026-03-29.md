## 📋 技术标准核心要点

### 设计准则 (The Core Principles)
1. **积木式模组化** - Lego-style独立模块
2. **下层不依赖上层** - Bottom-up Hierarchy
3. **故障隔离** - 单一模组故障不引发系统崩溃

### 业务链条 (8个标准化节点)
1. Ingest → 2. Cleaner → 3. Storer → 4. Analyzer → 5. Synthesizer → 6. Targeter → 7. Advisor → 8. Notary

### 网站三层架构
1. **逻辑层** (PHP) - SSR服务端渲染
2. **表现层** (CSS/HTML) - 与逻辑层解耦
3. **数据层** (多级) - 静态MD + 动态JSON

### 工业加固补丁 (V1.2.1)
1. **原子性写入协议**: Write(.tmp) → Validate → Rename(.json)
2. **日志分级轮转**: INFO/WARN/ERROR, 10MB上限, 7天保留
3. **双语调用契约**: 退出码(0=成功, 非0=重试/熔断)
4. **预留高速通道**: SQLite支持 (`database/sqlite/`)
5. **安全加固**: 敏感文件权限600, .gitignore覆盖.tmp/.bak

**核心功能设计**:
- **增量逻辑**: 基于 `fetch_time` 检查，避免重复数据入库
- **合并算法**: 
  - 日期存在 → 跳过（保持数据唯一性）
  - 日期不存在 → 追加到长期历史文件
  - 排序优化 → 按日期降序排列（最新的在前）
- **写前备份策略**: 修改长期历史文件前，在 `_PRIVATE_DATA/backups/` 下生成带时间戳的 `.bak` 文件
- **安全性加固**: 
  - 长期历史文件权限强制为 `600`（仅所有者可读写）
  - 备份文件同样设置严格权限
- **原子写入协议**: 遵循 Temp-Write → Validate → Rename 标准流程

**V1.2.1 工业规范实现**:
- ✅ 原子性写入协议: 临时文件 → 验证 → 原子重命名
- ✅ 错误契约: 定义 `[BACKUP_FAIL]`, `[PERMISSION_FAIL]`, `[MERGE_FAIL]` 等结构化错误码
- ✅ 调度器集成: 依赖关系 `ingest → cleaner → storer` 三级跳配置
- ✅ 安全加固: 文件权限控制、备份机制、敏感目录隔离
- ✅ 日志分级: INFO/WARN/ERROR 三级日志，结构化stderr输出
