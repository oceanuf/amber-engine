## 📋 技术标准核心要点

### 设计准则 (The Core Principles)
1. **积木式模组化** - Lego-style独立模块
2. **下层不依赖上层** - Bottom-up Hierarchy
3. **故障隔离** - 单一模组故障不引发系统崩溃

### 业务链条 (8个标准化节点)
1. Ingest → 2. Cleaner → 3. Storer → 4. Analyzer → 5. Synthesizer → 6. Targeter → 7. Advisor → 8. Notary

### 网站三层架构
1. **逻辑层** (PHP) - SSR服务端渲染
2. **表现层** (CSS/HTML) - 与逻辑层解耦
3. **数据层** (多级) - 静态MD + 动态JSON

### 工业加固补丁 (V1.2.1)
1. **原子性写入协议**: Write(.tmp) → Validate → Rename(.json)
2. **日志分级轮转**: INFO/WARN/ERROR, 10MB上限, 7天保留
3. **双语调用契约**: 退出码(0=成功, 非0=重试/熔断)
4. **预留高速通道**: SQLite支持 (`database/sqlite/`)
5. **安全加固**: 敏感文件权限600, .gitignore覆盖.tmp/.bak
